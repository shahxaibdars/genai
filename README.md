# Comparative Study: Fine-Tuning Methods for Stable Diffusion

## Project Overview

This repository contains a **comparative study of three fine-tuning methods** for adapting Stable Diffusion 1.5 to a specific food class (pizza) using varying dataset sizes. The study evaluates three popular techniques for domain adaptation with limited data:

1. **Textual Inversion (TI)** — Learn a new token embedding
2. **LoRA** — Low-Rank Adaptation of UNet attention layers  
3. **DreamBooth + LoRA** — Combines LoRA with prior preservation loss

All notebooks were **run on Google Colab with T4 GPU**.

## Experimental Setup

### Dataset
- **Source**: Food101 dataset
- **Class**: Pizza
- **Sizes**: 5, 10, and 20 images per training run
- **Resolution**: 512×512 pixels
- **Fixed seed**: Ensures consistency across all three methods

### Training Hyperparameters

| Parameter | Textual Inversion | LoRA | DreamBooth + LoRA |
|-----------|-------------------|------|-------------------|
| Learning Rate | 5e-4 | 1e-4 | 1e-4 |
| Batch Size | 1 | 1 | 1 |
| Max Steps | 3000 | 1000 | 1200 |
| Save Interval | 500 steps | 200 steps | 300 steps |
| Output Size | ~4 KB | ~3 MB | ~3 MB |

### Method Descriptions

#### 1. Textual Inversion (tex-inv-5.ipynb)

- **How it works**: Learns a new token embedding (~768 numbers) for the concept
- **Model frozen**: Yes, only token embedding is trainable
- **Output size**: ~4 KB (smallest)
- **Training speed**: Fast, but convergence slower (3000 steps needed)
- **Image quality**: Good but limited by frozen model

#### 2. LoRA (LoRA-5.ipynb)

- **How it works**: Injects small trainable rank-4 matrices into UNet attention layers
- **Trainable parameters**: ~1-4M (controlled via LoRA rank)
- **Model frozen**: Text encoder frozen, only UNet adapted
- **Output size**: ~3 MB
- **Training speed**: Very fast, converges in 1000 steps
- **Image quality**: Significantly better than Textual Inversion
- **Scalable**: Easy to adjust rank for quality/size trade-offs

#### 3. DreamBooth + LoRA (LoRA_Dreambooth-5.ipynb)

- **How it works**: LoRA with prior preservation loss
- **Prior preservation**: Uses ~200 class images generated by base model to prevent catastrophic forgetting
- **Unique token**: "sks" + class noun (e.g., "sks pizza") prevents semantic drift
- **Trainable parameters**: ~1-4M (same as LoRA)
- **Output size**: ~3 MB (same as LoRA)
- **Training speed**: Slower (1200 steps) due to prior loss computation
- **Image quality**: Highest quality among all three methods

## Evaluation Metrics

### CLIP Score
- Measures **semantic alignment** between generated image and text prompt
- Higher = better image-prompt correspondence
- Ranges from 0 to 1
- Three prompt complexity levels:
  - **Simple**: Single-word descriptions (e.g., "pizza")
  - **Medium**: 3-5 word descriptions (e.g., "delicious pepperoni pizza")
  - **Complex**: Full detailed descriptions with multiple attributes
- **Overall CLIP**: Macro average of simple/medium/complex

### DINO Score
- Measures **semantic consistency** of generated images
- Based on Vision Transformer features (DINOv2)
- Indicates whether the model learned consistent visual patterns for the concept
- Higher = more consistent style/appearance across generations
- Less sensitive to text alignment than CLIP

## Key Insights

### Training Efficiency
- **LoRA** achieves strong results with minimal training time (~7 min for T4)
- **DreamBooth+LoRA** trades 2-3x training time for improved quality
- **Textual Inversion** is slower despite smaller model (requires more steps to converge)

### Model Size
- **Textual Inversion**: ~4 KB (deployable on edge devices)
- **LoRA**: ~3 MB (excellent for distribution/fine-tuning)
- **DreamBooth+LoRA**: ~3 MB (same as LoRA)

### Image Quality
- **CLIP scores**: LoRA ≥ DreamBooth+LoRA > Textual Inversion
- **DINO scores**: LoRA > DreamBooth+LoRA > Textual Inversion
- Suggests LoRA achieves better semantic consistency with fewer resources

### Data Scaling
- All methods benefit from more training images
- Diminishing returns likely occur beyond 20 images for this pizza task

## How to Use Trained Models

### Load LoRA Weights
```python
from diffusers import StableDiffusionPipeline
from peft import PeftModel

base_model_id = "runwayml/stable-diffusion-v1-5"
lora_model_id = "<path-to-lora_final>"

pipe = StableDiffusionPipeline.from_pretrained(base_model_id)
pipe = PeftModel.from_pretrained(pipe.unet, lora_model_id)

# Generate image
image = pipe(prompt="a sks pizza on a plate").images[0]
```

### Load Textual Inversion Embeddings
```python
pipe = StableDiffusionPipeline.from_pretrained("runwayml/stable-diffusion-v1-5")
pipe.load_textual_inversion("<path-to-learned_embeds.safetensors>")

# Use learned token in prompts
image = pipe(prompt="a <food_style> pizza on a wooden table").images[0]
```
